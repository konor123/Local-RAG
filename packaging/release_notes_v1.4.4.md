## v1.4.4

### SQLite 인덱스 기본 사용
- SQLite metadata/FTS sidecar를 더 이상 메뉴 옵션으로 켜고 끄지 않고 기본 내부 기능으로 사용합니다.
- 기존 설정에서 SQLite가 꺼져 있던 경우에도 v1.4.4 기본 정책에 맞게 자동으로 활성화합니다.
- 트레이 메뉴에서 `SQLite 사이드카 사용`, `SQLite FTS 검색 사용` 항목을 제거했습니다.

### 인덱싱 상태 UI 개선
- 트레이 메뉴에 `📊 인덱싱 상태` 항목을 추가했습니다.
- 캐시 파일 수, 임베딩 상태, 현재 파일, 진행률, 성공/건너뜀/오류 수, 마지막 오류, backoff/disabled 상태를 확인할 수 있습니다.
- 확장자별 처리 통계와 SQLite/OCR 설정 상태도 같은 화면에서 확인할 수 있습니다.

### 직접 열람 OCR 자동화
- 직접 파일 열람 fallback에서 PDF 텍스트 추출이 실패하거나 내용이 너무 적으면 OCR을 자동으로 실행합니다.
- OCR 결과는 답변 컨텍스트에 포함되며 출처는 `ocr_direct_read`로 표시됩니다.
- OCR 처리 중에는 `OCR 엔진으로 읽는 중` 상태 로그를 출력해 사용자가 진행 상황을 알 수 있게 했습니다.

### OCR 엔진 preload
- 앱 시작 시 백그라운드에서 OCR 엔진을 미리 준비해 첫 OCR 실행 지연을 줄입니다.
- OCR preload 실패는 앱 실행을 막지 않으며, 실제 OCR이 필요할 때 다시 시도합니다.

### 제한값
- 직접 열람 OCR은 기본적으로 PDF당 최대 20페이지, 45초 제한을 둡니다.
- OCR이 실패하거나 시간이 초과되면 가능한 텍스트만 사용하고 상태 로그를 남깁니다.

### 검증
- SQLite 기본 활성화 및 기존 false 설정 migration 테스트를 추가했습니다.
- 직접 열람 PDF의 `ocr_needed` 결과가 자동 OCR로 보강되는지 테스트했습니다.
- 전체 테스트 스위트 기준 44개 테스트 통과, 5개 환경 의존 skip 상태입니다.
